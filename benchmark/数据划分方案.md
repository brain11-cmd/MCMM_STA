# 数据划分方案 (Data Split Scheme)

## 方案概述

本方案用于 GNN 模型训练的数据划分，采用**按温度划分**的策略，评估模型在跨温度条件下的泛化能力。

**核心原则：**
- 训练集和验证集使用 `vn40c` (-40°C) 和 `25c` (25°C) 的 corners
- 测试集使用所有 `125c` (125°C) 的 corners，评估跨温度泛化
- 3 个固定 anchors 用于多锚点融合，不在测试集中

---

## 数据范围

- **库类型**: RVT (仅 RVT)
- **总 Corners**: 27 个
  - Process: `ff` (Fast-Fast) / `ss` (Slow-Slow) / `tt` (Typical-Typical) = 3 种
  - Voltage: 3 种电压值
  - Temperature: 3 种温度 (`vn40c` / `25c` / `125c`)
- **Benchmarks**: 12 个（全部参与）
  - `aes`, `chameleon`, `dynamic_node`, `ethmac`, `fifo`, `gcd`, `jpeg`, `mock-alu`, `riscv32i`, `spi`, `tinyRocket`, `uart`

---

## Anchors (固定 3 个，且不在 Test 里)

用于多锚点融合的参考 corners，固定选择，不在测试集中：

- **A_fast**: `ff1p16vn40c`
- **A_typ**: `tt0p85v25c`
- **A_slow**: `ss0p7v25c`

> **注意**: 这 3 个 anchors 包含在 Train targets 中，作为训练数据的参考点。

---

## 划分方法

### 按温度划分 (By Temperature)

1. **Test targets (9 个)**: 所有 `125c` corners
   - 整档留出，用于测试跨温度泛化能力

2. **剩余 `vn40c` + `25c` 共 18 个**:
   - **Val targets (3 个)**: 从 18 个中抽取，覆盖 FF/SS/TT 各一个
   - **Train targets (15 个)**: 其余所有 `vn40c/25c` corners

---

## 详细划分列表

### 1. Test Targets (9 个: 全部 125°C)

用于最终测试，评估跨温度泛化能力：


用于模型选择和早停，覆盖不同 process corners：

```
ff0p95v25c

```

### 3. Train Targets (15 个: 其余所有 vn40c/25c)

训练集，按 process corner 分组：

#### FF (5 个)
```

ff1p16vn40c  (也是 Anchor A_fast)

```

#### SS (5 个)
```
ss0p7vn40c
ss0p7v25c    (也是 Anchor A_slow)
ss0p75v25c
ss0p95vn40c
ss0p95v25c
```

#### TT (5 个)
```
tt0p78vn40c
tt0p78v25c
tt0p85vn40c
tt0p85v25c   (也是 Anchor A_typ)
tt1p05vn40c
```

## 数据统计

### Corner 分配统计

| 集合 | 数量 | 占比 | 说明 |
|------|------|------|------|
| **Anchors** | 3 | 11.1% | 固定参考点（包含在 Train 中） |
| **Train** | 15 | 55.6% | 训练集 |
| **Val** | 3 | 11.1% | 验证集 |
| **Test** | 9 | 33.3% | 测试集 |
| **总计** | 27 | 100% | - |

### 训练数据量

- **Train corners**: 15 个
- **Benchmarks**: 12 个
- **总训练样本**: 15 × 12 = **180 个设计-corner 组合**

### PVT 覆盖分析

#### Train (15 个)
- **Process 覆盖**: FF(5) + SS(5) + TT(5) = 均衡分布
- **温度覆盖**: vn40c(8) + v25c(7) = 基本平衡

#### Val (3 个)
- **Process 覆盖**: FF(1) + SS(1) + TT(1) = 全面覆盖
- **温度覆盖**: vn40c(1) + v25c(2)

#### Test (9 个)
- **Process 覆盖**: FF(3) + SS(3) + TT(3) = 全面覆盖
- **温度**: 全部为 125°C（未见温度）

## 使用说明

### 1. 数据导出

对每个 benchmark，需要为以下 corners 运行 OpenTimer 并导出数据：

- **Anchors (3 个)**: 用于多锚点融合
- **Train (15 个)**: 训练数据
- **Val (3 个)**: 验证数据
- **Test (9 个)**: 测试数据

**总计**: 每个 benchmark 需要运行 27 个 corners（全部）

### 2. 训练时使用

- **训练**: 使用 Train (15 个) + Anchors (3 个，作为参考)
- **验证**: 使用 Val (3 个) 进行早停和模型选择
- **测试**: 使用 Test (9 个) 评估跨温度泛化

### 3. 数据组织建议

建议的目录结构：
```
data/
├── anchors/          # 3 个 anchor corners
├── train/            # 15 个 train corners
├── val/              # 3 个 val corners
└── test/             # 9 个 test corners
```

每个 corner 目录下包含：
- 图结构数据 (graph structure)
- 节点特征 (node features: arrival, slew, slack, etc.)
- 边特征 (edge features: arc delay, etc.)
- Endpoint 数据 (endpoint arrival/slack)

---

## 方案优势

1. ✅ **训练数据充足**: 180 个设计-corner 组合
2. ✅ **PVT 覆盖均衡**: 每个 process corner 都有 5 个训练样本
3. ✅ **泛化测试明确**: 测试集为未见温度（125°C）
4. ✅ **验证集有代表性**: 覆盖 FF/SS/TT 各一个
5. ✅ **Anchor 选择合理**: 覆盖 fast/typical/slow，且不在测试集中

---

## 注意事项

1. **验证集较小**: Val 只有 3 个 corners，如果验证不稳定，可以考虑：
   - 从 Train 中再选 1-2 个加入 Val
   - 或使用交叉验证

2. **跨设计泛化**: 当前方案只评估跨温度泛化，如需评估跨设计泛化，可在 Phase 2 进行 8/4 benchmark split

3. **数据导出**: 需要确保所有 27 个 corners 的数据都已正确导出
